# چک‌لیست Go / No-Go قبل از شروع ساخت MVP

تا وقتی موارد زیر روشن نشده، شروع کدنویسی کامل MVP ریسک دوباره‌کاری دارد.

## 1. دامنه MVP

- [ ] فاز اول فقط محصول‌محور است.
- [ ] دسته‌های شروع محدود و قطعی شده‌اند: موبایل، لپ‌تاپ/کامپیوتر، لوازم خانگی دیجیتال/برقی، صوتی‌تصویری، گجت هوشمند.
- [ ] رستوران، املاک، خدمات، پزشک، آموزشگاه و کسب‌وکارهای محلی به فاز دوم منتقل شده‌اند.
- [ ] Checkout داخلی فقط برای فروشندگان بدون سایت یا فروشندگانی که خرید امن بیزبورد را فعال می‌کنند ساخته می‌شود.
- [ ] لینک خروجی برای فروشندگان دارای سایت از ابتدا track می‌شود.

## 2. دیتابیس و معماری

- [ ] PostgreSQL انتخاب قطعی است.
- [ ] مدل postmeta/key-value وردپرسی ممنوع است.
- [ ] schemaهای دامنه‌ای auth، directory، catalog، commerce، finance، ai، crawler، seo، reviews، media، admin ساخته می‌شوند.
- [ ] product و offer جدا هستند.
- [ ] business و mini-site جدا ولی مرتبط هستند.
- [ ] ledger دوبل-entry برای پول طراحی شده است.
- [ ] migration اولیه قبل از کدنویسی اجرا و review شده است.

## 3. AI-Buildable بودن پروژه

- [ ] AI_CONTEXT.md آماده است.
- [ ] AI_CODING_RULES.md آماده است.
- [ ] AI_FEATURE_TEMPLATE.md آماده است.
- [ ] هیچ فیچری بدون Technical Plan، migration، OpenAPI و تست اجرا نمی‌شود.
- [ ] AI اجازه تغییرات وسیع بی‌دلیل در فایل‌ها ندارد.

## 4. پنل ادمین و کسب‌وکار

- [ ] Scope پنل ادمین MVP مشخص است.
- [ ] Scope پنل کسب‌وکار MVP مشخص است.
- [ ] نقش‌ها و سطح دسترسی‌ها مشخص شده‌اند.
- [ ] Audit log برای عملیات حساس در نظر گرفته شده است.

## 5. پرداخت و مالی

- [ ] زرین‌پال و زیبال به عنوان gatewayهای اولیه انتخاب شده‌اند.
- [ ] verify سمت سرور الزامی است.
- [ ] callback پرداخت idempotent است.
- [ ] ledger قبل از checkout ساخته می‌شود، نه بعداً.
- [ ] تسویه بدون KYC/KYB انجام نمی‌شود.
- [ ] dispute/refund حداقل در سطح MVP تعریف شده است.

## 6. سئو و کرالر

- [ ] robots.txt و سیاست crawl قانونی مشخص است.
- [ ] صفحه AI-generated بدون quality gate ایندکس نمی‌شود.
- [ ] صفحات فیلتر بی‌ارزش noindex هستند.
- [ ] URLهای اصلی قطعی شده‌اند.
- [ ] schema JSON-LD برای Product، Offer، Breadcrumb و LocalBusiness در مسیر طراحی قرار دارد.

## 7. امنیت و حریم خصوصی

- [ ] env و secretها داخل repo ذخیره نمی‌شوند.
- [ ] rate limit برای login، search، chat، payment callback وجود دارد.
- [ ] اطلاعات کارت بانکی ذخیره نمی‌شود.
- [ ] IP در analytics/ads hash می‌شود.
- [ ] قوانین استفاده، حریم خصوصی، سیاست AI chat و قوانین فروشنده تهیه می‌شود.

## نتیجه

- [ ] GO: همه موارد حیاتی تایید شده‌اند.
- [ ] NO-GO: موارد حیاتی ناقص هستند و نباید MVP ساخته شود.
