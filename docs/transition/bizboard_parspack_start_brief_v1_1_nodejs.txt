مستند انتقال پروژه بیزبورد برای شروع مستقیم روی پارس‌پک
نسخه: 1.1
هدف: بردن کل کانتکست این چت به یک چت جدید و شروع اجرای پروژه از صفر روی پارس‌پک، بدون کار روی سیستم شخصی
تاریخ: 2026-05-13
به‌روزرسانی 1.1: تاکید صریح روی Node.js/NestJS به‌عنوان بک‌اند اصلی و شروع مستقیم روی پارس‌پک

====================================================================
1) وضعیت فعلی پروژه
====================================================================
کاربر هنوز هیچ کاری روی سیستم خودش انجام نداده است.
مسیر جدید پروژه این است:
- توسعه و اجرای اولیه مستقیم روی زیرساخت پارس‌پک انجام شود.
- فعلاً نباید دستورهای local-first مثل نصب روی لپ‌تاپ، Docker Desktop، اجرای localhost و Cursor روی سیستم شخصی محور کار باشد.
- بهتر است از یک Git repository، سرور/محیط پارس‌پک، Docker/Docker Compose، PostgreSQL، Redis و Object Storage استفاده شود.
- اگر لازم شد کد تولید شود، باید به شکل monorepo آماده‌ی deploy روی پارس‌پک باشد.

این مستند خلاصه کامل تصمیم‌های قبلی، معماری، دیتابیس، AI، پنل‌ها و مسیر شروع روی پارس‌پک است.

تصمیم قطعی تکنولوژی بک‌اند:
- بک‌اند اصلی پروژه حتماً Node.js است.
- زبان بک‌اند TypeScript است.
- فریم‌ورک بک‌اند NestJS است.
- اپلیکیشن API در مسیر apps/api ساخته می‌شود.
- Workerها، crawlerها، job queue و AI jobs هم با Node.js/TypeScript ساخته می‌شوند.
- پروژه PHP، WordPress، Laravel، MySQL یا ساختار وردپرسی نیست.
- هدف استقرار مستقیم روی پارس‌پک با Docker/Docker Compose است، نه اجرای local-first روی سیستم شخصی.


====================================================================
2) خلاصه محصول بیزبورد
====================================================================
بیزبورد قرار است از یک دایرکتوری ساده کسب‌وکار به یک «موتور تصمیم‌گیری، مقایسه و خرید هوشمند» تبدیل شود.

هدف برای کاربر:
- راحت‌تر محصول، فروشگاه، خدمت یا کسب‌وکار مناسب را پیدا کند.
- با دستیار هوش مصنوعی وسط صفحه صحبت کند.
- محصول‌ها، قیمت‌ها، فروشنده‌ها، نقدها و اعتبار فروشنده‌ها را مقایسه کند.
- اگر فروشنده سایت دارد، به سایت فروشنده برود.
- اگر فروشنده سایت ندارد، از مینی‌سایت بیزبورد خرید کند.

هدف برای کسب‌وکار:
- مشتری بگیرد.
- صفحه/مینی‌سایت اختصاصی داشته باشد.
- محصول و خدمت وارد کند.
- برای سئو و AI بیزبورد صفحه خودش را بهینه کند.
- تبلیغ کلیکی یا اسپانسر بگیرد.
- اگر سایت ندارد، از طریق بیزبورد سفارش و پرداخت بگیرد.

هسته نسخه قدیمی در پیچ‌دک:
- مشکل ۱: سختی انتخاب و تصمیم‌گیری برای مردم.
- مشکل ۲: هزینه بالای تبلیغات برای کسب‌وکارها.
- راه‌حل قدیمی: اپ/سایت نقد و بررسی و امتیازدهی کسب‌وکارها.

نسخه جدید:
- AI-first
- Product-first در فاز اول
- SEO-first
- Marketplace-ready
- Crawler/Data ingestion-ready
- قابل توسعه با AI programming

5) تصمیم‌های فنی اصلی
====================================================================
Backend اصلی پروژه:
- Node.js به‌عنوان runtime اصلی بک‌اند
- TypeScript برای کل بک‌اند
- NestJS به‌عنوان فریم‌ورک API
- REST API با Swagger/OpenAPI
- Workerها با Node.js + BullMQ
- Crawler اولیه با Node.js/TypeScript
- AI Provider Gateway با Node.js و SDK/API سازگار با OpenAI
- معماری Modular Monolith در شروع، قابل جدا شدن به سرویس‌های مستقل در آینده
- ممنوع: PHP/WordPress/Laravel برای هسته پروژه
- ممنوع: MySQL و ساختار wp_posts/wp_postmeta

Frontend:
- Next.js + React + TypeScript
- SSR/SSG/ISR برای سئو
- Tailwind CSS
- RTL و فارسی‌سازی کامل

Database:
- PostgreSQL
- نه MySQL
- نه ساختار وردپرسی
- نه wp_posts/wp_postmeta/wp_terms
- افزونه‌های پیشنهادی:
  - pgcrypto برای UUID
  - pg_trgm برای سرچ فازی و فارسی
  - pgvector برای embedding و semantic search
  - PostGIS برای لوکیشن و نزدیک من
  - btree_gin / btree_gist برای ایندکس‌های ترکیبی

Cache/Queue:
- Redis
- BullMQ برای job queue، crawler، AI jobs، SEO jobs، import jobs

Search در MVP:
- PostgreSQL Full Text Search
- pg_trgm
- pgvector
بعد از رشد:
- OpenSearch / Elasticsearch / Meilisearch در صورت نیاز

Object Storage:
- فضای ابری S3-compatible پارس‌پک برای عکس محصول، لوگو، گالری، فایل‌های feed، خروجی crawler، backup سبک

Deployment:
- مستقیم روی پارس‌پک
- گزینه پیشنهادی برای شروع: سرور ابری پارس‌پک + Docker Compose
- در صورت استفاده از PaaS پارس‌پک، باید سرویس‌ها جدا تعریف شوند: web، api، worker، crawler، postgres، redis یا دیتابیس/ردیس جدا

Monitoring:
- Sentry برای خطا
- لاگ ساختارمند JSON
- Health check
- در آینده Prometheus/Grafana یا سرویس مانیتورینگ

10) AI Assistant و Provider Gateway
====================================================================
بخش AI باید provider-agnostic باشد.
یعنی منطق بیزبورد نباید به GapGPT، OpenAI، DeepSeek یا هیچ سرویس خاصی قفل شود.

وضعیت فعلی موردنظر:
- فعلاً استفاده از GapGPT
- بعداً امکان اضافه کردن OpenAI
- بعداً امکان اضافه کردن DeepSeek
- بعداً امکان اضافه کردن هر API سازگار با OpenAI API یا providerهای دیگر

هیچ جای پروژه مستقیم OpenAI/GapGPT/DeepSeek را صدا نزند.

نکات امنیتی AI:
- API keyها فقط در server/env باشند.
- هیچ کلیدی وارد frontend نشود.
- request id، latency، usage، provider، model و status لاگ شود.
- fallback فعلاً خاموش باشد؛ چون ممکن است داده کاربر بدون اطلاع به provider دیگر برود.
- خروجی JSON دستیار خرید باید validate شود.
- AI نباید قیمت یا موجودی نامطمئن را قطعی اعلام کند.
- تبلیغ و پیشنهاد ارگانیک باید جدا و شفاف باشند.

14) SEO و crawler
====================================================================
هدف SEO:
برای هر intent مهم، صفحه‌ای مفید، سریع، ساختاریافته و قابل crawl ساخته شود.

هیچ کس نمی‌تواند رتبه ۱ گوگل را تضمین کند، ولی باید معماری طوری باشد که شانس رتبه‌گیری بالا برود و ریسک پنالتی کم شود.

قالب صفحات SEO:
- دسته محصول
- برند + دسته
- محصول canonical
- مقایسه محصول‌ها
- بهترین‌ها مثل «بهترین گوشی تا ۳۰ میلیون»
- فروشگاه معتبر
- صفحه محلی در فاز دوم

Crawler/Data ingestion:
- قرار نیست بدون مجوز کل اینترنت کپی شود.
- باید robots.txt، قوانین سایت‌ها و قراردادها رعایت شود.
- مسیرهای ورودی دیتا:
  - Merchant feed
  - API فروشگاه
  - CSV/XML/JSON
  - sitemap crawl با مجوز
  - JSON-LD extraction
  - Manual entry
  - Admin import
  - Claim & correction

کیفیت دیتا:
- raw data مستقیم publish نشود.
- quality_score لازم است.
- محصول تکراری باید merge شود.
- قیمت باید last_seen_at داشته باشد.
- صفحات ضعیف یا تکراری noindex شوند.

15) مسیر شروع مستقیم روی پارس‌پک
====================================================================
چون کاربر می‌خواهد مستقیم روی پارس‌پک شروع کند، مسیر پیشنهادی این است:

گزینه پیشنهادی برای MVP:
- یک سرور ابری پارس‌پک برای staging
- Docker + Docker Compose روی همان سرور
- Git repository برای کد
- دامنه/subdomain برای staging
- PostgreSQL و Redis داخل Docker در شروع، یا سرویس جدا اگر پارس‌پک/زیرساخت انتخابی آماده دارد
- Object Storage پارس‌پک برای فایل‌ها
- SSL با Caddy/Nginx/Reverse proxy

دامنه‌های پیشنهادی staging:
- staging.bizboard.co برای frontend
- api-staging.bizboard.co برای backend
- admin-staging.bizboard.co فقط اگر admin جدا شد؛ در شروع لازم نیست

سرویس‌های Docker Compose روی پارس‌پک:
- web: Next.js روی Node.js
- api: NestJS API روی Node.js
- worker: Node.js BullMQ workers
- crawler: Node.js crawler workers
- postgres: PostgreSQL
- redis: Redis
- reverse-proxy: Caddy یا Nginx

نکته مهم:
همه سرویس‌های application-level یعنی web، api، worker و crawler با Node.js اجرا می‌شوند. تنها سرویس‌های غیر Node.js دیتابیس، Redis و reverse proxy هستند.

اولین خروجی قابل قبول روی پارس‌پک:
- صفحه web باز شود.
- api health جواب دهد.
- Swagger باز شود.
- دیتابیس migrate شده باشد.
- seed دسته‌بندی‌ها انجام شده باشد.
- صفحه اصلی AI-first با چت mock باز شود.
- چند محصول، فروشنده و offer نمونه دیده شود.
- envها امن و جدا از repo باشند.

URLهای هدف:
- https://staging.bizboard.co
- https://api-staging.bizboard.co/api/v1/health
- https://api-staging.bizboard.co/api/docs

17) چک‌لیست Go/No-Go قبل از لانچ staging
====================================================================
Go اگر:
- deploy روی پارس‌پک پایدار است.
- DB migration تکرارپذیر است.
- backup اولیه تعریف شده است.
- envها در repo نیستند.
- API health و Swagger کار می‌کند.
- seed data انجام شده است.
- صفحه اصلی، دسته، محصول، کسب‌وکار باز می‌شوند.
- لاگ خطا وجود دارد.
- AI key فقط backend است.
- حداقل security headers فعال است.

No-Go اگر:
- secret داخل repo است.
- دیتابیس با ساختار وردپرسی/متا ساخته شده است.
- AI مستقیم از frontend صدا زده می‌شود.
- payment بدون verify طراحی شده است.
- migrationها دستی و غیرقابل تکرارند.
- لاگ و health check وجود ندارد.
- صفحات محصول canonical ندارند.

18) پرامپت آماده برای چت جدید
====================================================================
این متن را در چت جدید بده و همین فایل را هم upload کن:

«این مستند انتقال پروژه بیزبورد است. من می‌خواهم پروژه را از صفر مستقیم روی پارس‌پک شروع کنم، نه روی سیستم خودم.

لطفاً اول کل مستند را بخوان و بعد یک پلن اجرایی بده برای اینکه روی پارس‌پک staging را بالا بیاوریم.

قوانین:
1. مسیر local-first نده؛ من فعلاً روی سیستم خودم کاری نکرده‌ام.
2. هدف اول ساخت staging روی پارس‌پک است.
3. پروژه باید monorepo باشد: Next.js frontend روی Node.js، NestJS backend روی Node.js/TypeScript، PostgreSQL، Redis، Node.js worker، Node.js crawler.
4. دیتابیس باید PostgreSQL غیروردپرسی باشد؛ wp_posts/wp_postmeta ممنوع.
5. AI باید provider-agnostic باشد؛ فعلاً GapGPT default باشد و بعداً OpenAI و DeepSeek اضافه شوند.
6. هیچ API key نباید وارد frontend یا repo شود.
7. قبل از کدنویسی Technical Plan بده.
8. فایل‌هایی که باید بسازی را لیست کن.
9. بعد از پلن، اسکلت Node.js deploy-ready برای پارس‌پک بساز: Dockerfileهای web/api/worker/crawler، docker-compose.staging.yml، env.example، README deployment، health check، Swagger، migration اولیه، seed اولیه، صفحه اصلی mock.
10. اگر جایی به اطلاعات حساب پارس‌پک، دامنه یا secret نیاز داشتی، فقط اسم متغیرها را بگو و مقدار واقعی را از من نخواه داخل کد بنویسم.»
